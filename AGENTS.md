# AGENTS.md

This file provides context and instructions for AI agents working on the `sql2js` project.

## Project Overview

`sql2js` is an SQL-to-JavaScript compiler for JSON data, featuring an interactive Terminal User Interface (TUI). It allows users to write SQL-like queries to filter, sort, transform, and aggregate data within JSON files.
The compilation pipeline is as follows:
`SQL query → Lexer (ANTLR4) → Parser (ANTLR4) → AST Builder → Semantic Analyzer → Code Generator (JavaScript) → Runtime Executor`

## Technology Stack
*   **Language:** JavaScript (ESM - ECMAScript Modules)
*   **Runtime:** Node.js (≥ 18)
*   **Parser Generator:** ANTLR4 (`antlr4` npm package for runtime)
*   **TUI Framework:** Ink (React for terminals, but using `React.createElement` instead of JSX to avoid build steps)
*   **Styling:** Chalk (for terminal colors)

## Key Directories & Files
*   `grammar/JsonQuery.g4`: The ANTLR4 grammar definition. **CRITICAL:** Operator precedence in this grammar is ordered from highest to lowest (first alternative is highest precedence), which is standard for ANTLR4 left-recursive rules.
*   `src/generated/`: Contains files auto-generated by ANTLR4. **Do not edit these manually.**
*   `src/ast/`: Contains custom AST node definitions (`nodes.js`) and the visitor that builds them from the parse tree (`builder.js`).
*   `src/semantic/analyzer.js`: Performs semantic validation on the AST (e.g., checking aliases, valid aggregate usage).
*   `src/codegen/generator.js`: Translates the validated AST into executable JavaScript code.
*   `src/runtime/executor.js`: Executes the generated JavaScript against the provided JSON data.
*   `src/tui/app.js`: The main TUI application. Uses `React.createElement` instead of JSX.
*   `src/pipeline.js`: Orchestrates the entire compilation pipeline.
*   `src/index.js`: The main CLI entry point.
*   `data/`: Contains sample JSON data files (`users.json`, `orders.json`, `customers.json`) used for testing and demonstration.

## Operational Commands
*   **Install dependencies:** `npm install`
*   **Generate parser:** `npm run generate` (Note: Requires Java to be installed, as it runs the ANTLR4 jar). Run this anytime `JsonQuery.g4` is modified.
*   **Run TUI:** `npm start` (or `node src/index.js`)
*   **Run tests:** `node src/test.js`

## Coding Conventions & Guidelines
1.  **ES Modules:** Use `import` and `export` exclusively. Do not use `require()`. Ensure file extensions (`.js`) are included in relative imports.
2.  **No Build Step for TUI:** The TUI in `src/tui/app.js` is built with Ink/React, but it is written using standard JavaScript (`React.createElement`) rather than JSX. This is a deliberate choice to avoid needing Babel or a bundler. **Do not introduce JSX.**
3.  **Error Handling:** The compiler uses a custom `CompilerError` class (in `src/errors/errors.js`). The semantic analyzer is designed to collect *all* errors, rather than failing fast on the first error.
4.  **Codegen Security:** The `Executor` uses `new Function()` instead of `eval()` to run the generated code in a slightly more isolated scope.
5.  **Grammar Changes:** If you modify `grammar/JsonQuery.g4`, you **must** run `npm run generate` and ensure the generated files are placed correctly in `src/generated/` (and not nested in a subdirectory like `src/generated/grammar/`).
6.  **Code Style:** Keep code clean and well-commented. Use camelCase for variables and functions, PascalCase for classes and AST nodes.

## Constraints
*   Do not add a build step (Webpack, Babel, Vite, etc.) to this project. It is designed to run directly with Node.js.
*   Do not modify the `src/generated/` files directly. Always modify the `.g4` grammar and regenerate.
